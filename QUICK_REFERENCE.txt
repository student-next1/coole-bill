❓ PERTANYAAN: "search-card.blade.php itu tampil di bagian mana kalau di web?"

✅ JAWABAN CEPAT:

   search-card.blade.php tampil di halaman untuk MENCARI DAN MEMILIH KARTU
   setelah user memilih metode pembayaran "🆔 KARTU ID"

   📍 LOKASI DALAM FLOW:
   
       Halaman: transaksi/create
       (Pilih produk + pilih metode pembayaran)
              ↓
              [User klik "🆔 KARTU ID"]
              ↓
       🎯 Halaman: transaksi/search-card ← INI HALAMAN YANG DITANYA!
       (Cari kartu, pilih kartu)
              ↓
              [User klik "✓ Gunakan Kartu Ini"]
              ↓
       Halaman: transaksi/invoice
       (Edit struk)


═══════════════════════════════════════════════════════════════════════════════

🔍 APA ITU search-card.blade.php?

   Halaman untuk search dan pilih kartu pembayaran
   
   User bisa:
   ✓ Search kartu by username (contoh: card_001)
   ✓ Search kartu by card_code (contoh: CARD-202607...)
   ✓ Search kartu by barcode
   ✓ Pilih kartu yang mau digunakan
   
   Kartu yang cocok ditampilkan dengan detail:
   - Nama pemilik
   - Username
   - Saldo
   - Barcode
   - Status (Aktif/Nonaktif)

═══════════════════════════════════════════════════════════════════════════════

📋 FLOW DETAILS:

   STEP 1: transaksi/create
           User pilih produk → masuk ke keranjang
   
   STEP 2: User klik "🆔 KARTU ID" button
           ↓
           POST /transaksi/select-payment
           ↓
           Server redirect ke /transaksi/search-card
   
   STEP 3: search-card.blade.php TAMPIL
           User search kartu, lihat hasil, pilih kartu
           ↓
           POST /transaksi/select-card dengan card_id
           ↓
           Server redirect ke /transaksi/invoice
   
   STEP 4: invoice.blade.php TAMPIL
           User edit struk, confirm
           ↓
           POST /transaksi/confirm
           ↓
           Server redirect ke /transaksi/confirmation
   
   STEP 5: confirmation.blade.php TAMPIL
           Success page dengan struk

📝 KESIMPULAN:

   search-card.blade.php adalah halaman INTERMEDIARY (perantara)

   Memungkinkan user:
   1. Mencari kartu dengan berbagai kriteria
   2. Memilih kartu yang akan digunakan
   3. Melanjutkan ke halaman edit struk
